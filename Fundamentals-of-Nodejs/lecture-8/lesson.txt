Learning about import.meta object.

-> import.meta is a special object available inside ES modules (ECMAScript modules).
It contains useful metadata (information) about the current module.

You can access it only when using ES6 modules, i.e., files loaded using:

import ...

1Ô∏è‚É£ import.meta.url

Gives the absolute URL to the current module file.

Very useful for determining file paths inside modules.

Example:

console.log(import.meta.url);


Output example:

file:///home/user/project/module.js


In the browser it gives the web URL, like:

https://example.com/scripts/module.js

üß† Why do we need import.meta?

Because ES modules do not have values like __dirname or __filename, which are available in CommonJS (require system).

Example workaround in Node.js:

import { fileURLToPath } from 'url';
import { dirname } from 'path';

const __filename = fileURLToPath(import.meta.url);
const __dirname = dirname(__filename);

console.log(__dirname);
